# OCKey Windows 使用说明

本地运行一个 OpenAI 兼容接口，把 OpenCode 的免费模型暴露成 `ockey_...` 形式的 Key，供其它工具调用。
服务地址固定为 `http://127.0.0.1:8789`。

---

## 准备情况

1. **Node.js**：已安装（v24）。✅
2. **依赖**：已安装（`npm install` 完成）。✅
3. **OpenCode 运行时**：已改为**自动识别**你电脑上已安装的 OpenCode，无需手动复制。
   程序会按以下顺序查找 CLI：
   - 环境变量 `OCKEY_OPENCODE_PATH` 指定的路径
   - `%LOCALAPPDATA%\OpenCode\opencode-cli.exe`（即你当前安装的位置）✅
   - `%LOCALAPPDATA%\OpenCode\opencode.exe`
   - 都没有，才回退到 `resources\runtime\bin\opencode-cli.exe`

   > 如果以后 OpenCode 装到了别处，在启动前设置环境变量即可：
   > ```powershell
   > $env:OCKEY_OPENCODE_PATH="完整路径\opencode-cli.exe"
   > ```

---

## 日常启动

```powershell
cd D:\OCKEY\OCKey-Windows
npm run start:server
```

看到 `OCKey Windows server running at http://127.0.0.1:8789` 即成功。
浏览器打开 **http://127.0.0.1:8789** 进入控制台。

> 想要系统托盘图标版，改用 `npm start`。

---

## 控制台操作

1. 打开 http://127.0.0.1:8789
2. OpenCode 未登录时，点 **「开始登录」**，在弹出的命令行窗口完成 OpenCode 官方登录。
3. 点 **「生成新通道」**，系统会自动把 `Key + URL` 复制到剪贴板。
4. 在你的其它工具里填入：
   ```
   OPENAI_BASE_URL=http://127.0.0.1:8789/v1
   OPENAI_API_KEY=ockey_xxx
   ```
5. 「测试」可检查单个通道；「测试全部通道」会逐个调用模型（消耗额度）。

## 常见问题

**启动报 `EADDRINUSE: address already in use 127.0.0.1:8789`**
8789 端口已被占用——通常是之前的服务还在后台跑。
- 先打开 http://127.0.0.1:8789，能用就直接用。
- 要重启，先结束占用进程：
  ```powershell
  Get-NetTCPConnection -LocalPort 8789 -State Listen | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
  ```
  再 `npm run start:server`。

**验证服务**
```powershell
curl http://127.0.0.1:8789/health
```
返回的 JSON 里 `opencodeStatus.available` 为 `true`、能看到 `path` 指向你的 opencode-cli.exe，即接入成功。

**停止服务**：在运行服务的终端按 `Ctrl + C`。

## 接口一览

- `GET  /health`               健康检查（无需 Key）
- `GET  /v1/models`            模型列表
- `POST /v1/chat/completions`  对话补全
- `POST /v1/responses`         Responses 接口

`/v1/*` 需请求头 `Authorization: Bearer ockey_...`。当前 1.0 版**不支持** `stream=true`。

## 打包成 EXE

项目已配好打包工具，**双击 `打包EXE.bat`** 即可（或命令行 `npm run build`）。
打包前会自动运行 `npm run prepare:runtime`，只复制 OpenCode CLI，不打包 OpenCode 桌面应用。
注意：必须在 Windows 上打包；首次会自动下载 Electron 和 OpenCode CLI 运行时，需耐心等待。

打包产物在 `dist\` 目录：

- **便携版（免安装，双击即用）**：`dist\OCKey-Portable-1.0.0-Windows.exe`
- **安装版（带快捷方式）**：`dist\OCKey-Setup-1.0.0-Windows.exe`

生成的 exe 内置 Node 运行时和独立 OpenCode CLI。若要改用自己从 OpenCode 源码构建的 CLI，请先设置 `OCKEY_OPENCODE_RUNTIME_SOURCE`，再运行准备脚本：

```powershell
$env:OCKEY_OPENCODE_RUNTIME_SOURCE="完整路径\opencode.exe"
npm run prepare:runtime
resources\runtime\bin\opencode-cli.exe --version
```

如果版本命令不能输出，请先替换为可用的 OpenCode CLI，或让目标电脑安装 OpenCode，由 OCKey 自动识别。
双击 exe 后会出现系统托盘图标，从托盘进入控制台。
